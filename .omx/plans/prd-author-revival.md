# PRD: Author Revival First-Stage Pivot

## Goal

把系统主线从“V2 长篇连载规划引擎”切到“作者复活引擎”的第一阶段：红楼专用验证版。成功标准不是多章批量续写，而是生成章节能进入机械闸门和 1000 字盲测流程。

## Scope

本阶段交付：

1. 作品工作区模型与本地产物生成。
2. 红楼输入文本的章节切分。
3. `style_bible`、禁词表、人物声口卡、文体指标。
4. 扩展机械风格闸门。
5. 1000 字盲测挑战数据结构：生成片段 + 3 段原文片段 + 隐藏标签。
6. 路线图更新，明确 V2 被降级为辅助层。
7. 单元测试覆盖上述行为。

本阶段不交付：

- 完整 300 万字规模化。
- 自动人物卡全量抽取的高召回版本。
- Web 工作台大改版。
- 强依赖真实在线 GPT-5.5 的端到端判别闭环。

## User Stories

### US-001: 红楼文本可切分为章节

作为用户，我希望系统能从已有红楼文本识别章节边界，以便后续第 81 回验证能引用前文片段。

Acceptance Criteria:

- 给定包含“第八十回”等章回标题的文本，能输出有序章节列表。
- 每章包含编号、标题、正文、字符范围。
- 对无法识别章回标题的文本，退化为单章而不是崩溃。

### US-002: 系统能生成作品声纹手册

作为用户，我希望系统建立 `style_bible`，以便正文生成不只是剧情续写，而能贴近原作者文气。

Acceptance Criteria:

- 输出叙事 pattern、常见虚词指标、平均句长、对白比例。
- 输出禁用现代词表。
- 输出至少一组人物声口卡的结构。
- 产物可 JSON/Markdown 落盘。

### US-003: 机械闸门能拦低级不像原文的问题

作为用户，我希望系统在盲测前先拦截明显 AI 包装语、现代分析词和文体偏移。

Acceptance Criteria:

- 拦截“以下是”“总结”“作为 AI”等包装语。
- 拦截“安全感”“情绪价值”“心理压力”等现代抽象词。
- 可根据 style metrics 检查句长、对白比例、常见虚词密度偏移。
- 检查繁简混杂。

### US-004: 盲测挑战包含生成段与原文对照段

作为用户，我希望盲测器能构造 4 段隐藏来源的片段，以便判别生成段是否容易被识别。

Acceptance Criteria:

- 从生成正文截取约 1000 中文字。
- 从原文章节中抽取 3 段约 1000 中文字。
- 输出乱序片段，不直接暴露哪段是生成文。
- 保留内部 answer key 供后续判别或人工评分。

## Risks

- 红楼文本标题格式可能不稳定：使用多种中文数字标题模式并提供单章 fallback。
- 机械指标可能误伤短文本：短文本只给 warning，长文本再 fail。
- 旧 V2 代码仍被 Web/CLI 依赖：第一阶段只新增/扩展 revival 组件，不删除旧链路。

## Delivery Definition

- 新路线图存在并描述执行顺序。
- 新模型与服务有测试。
- 现有相关测试通过。
- `pytest` 尽可能全量通过；如环境或在线模型导致失败，需要记录具体失败原因。

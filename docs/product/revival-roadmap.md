# TaiJianICU Revival 路线图

## 1. 决策

从现在开始，产品主线切换为“作者复活引擎”。

旧 V2 的世界层、规划层和参考层不删除，但降级为辅助能力；它们不再定义成功标准。新的成功标准是：

- 盲看 1000 字不容易识别为 AI 续写。
- 人物声口稳定。
- 不出现现代分析腔、AI 包装语和解释性总结。
- 长篇硬事实不乱。
- 用户导演意图能被作品内部化，而不是原样进入正文。

## 2. 当前阶段

当前第一目标是可信评审：让 Studio 成为能证明“像不像原作者”、并能指导修订的桌面工作台。

第 81 回已经作为“文风明显跑偏”的单章修复样本进入 gate / rewrite 闭环；验证扩展到第 120 回和后段连续章节。现在的产品判断不再只看能不能生成，而是看 `trust_report.json`、盲测、clean/source voice gate、manifest 状态和 `benchmark-multi` 连续摘要是否共同支持交付。

## 3. 第一阶段交付

1. 文本清洗与章节切分。
2. 红楼 `style_bible`。
3. 人物声口卡：宝玉、黛玉、宝钗、袭人、王夫人、凤姐、迎春、香菱。
4. `forbidden_words`。
5. 机械风格闸门。
6. 1000 字盲测挑战。
7. 盲测判别器与失败重写循环。
8. 第 81 回单章重写流程。
9. Source-voice gate：用源文本章节长度、繁简一致性、解释性抒情腔和声口统计拦截写回。
10. 第 120 回末段回归：复用前 80 回索引，验证短章会进入修订循环，最终达到 `4305/4257` 中文字符并通过 gate。
11. 近章重复护栏：连续章节若复用相同开头或转场，写回前进入最多 3 轮定向修订；修订提示必须要求重写当前章前两段并换入场焦点。
12. `trust_report.json`：一等可信报告，只展示总状态和分项证据，不生成百分制总分。
13. `director_constraints.json`：导演意图内部化，优先 LLM 结构化转换，失败时规则兜底，Studio 可编辑并在选择人物走向时锁定快照。
14. `benchmark-multi` 连续可信回归：每章输出长度、叙述节奏、对白比例、clean prose、重复和 overall issue 摘要，并给出 `revival_status`。
15. Studio 单章评审：可信报告按 `fail -> warning -> not_ready -> pass` 排序，把“优先处理”“检查明细”和下一步修订动作分开展示。

## 4. 旧 V2 的处理

保留：

- `LiteLLMService`
- session 存储
- Web/API 基础结构
- 已有 quality / benchmark 基础设施
- 已有 V2 模型，作为可选辅助状态

暂停作为主线：

- 重型 LightRAG 图谱驱动
- `WorldModel -> ArcOutline -> ChapterBrief` 的默认主流程
- 无质量门兜底的后 40 回批量生成

## 5. 后续阶段

第二阶段：通用断更小说版。

- 自动章节切分。
- 自动人物卡。
- 自动伏笔队列。
- 用户导演输入内部化。
- 多轮盲测重写。

第三阶段：长篇规模化。

- 支持 300 万字。
- 支持卷层记忆。
- 支持最近 20 章窗口。
- 支持人物出场检索。
- 支持成本预估。

## 6. 验证

每阶段必须有：

- 单元测试。
- 本地烟测。
- 至少一个真实文本样本。
- 明确失败样本与下一轮修正依据。

当前可复现验证不放在路线图里滚动维护，避免路线图变成运行日志。最新命令、样本和缺口见 [当前验证状态](../engineering/revival-validation-status.md)。

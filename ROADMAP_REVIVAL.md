# TaiJianICU Revival 路线图

## 1. 决策

从现在开始，产品主线切换为“作者复活引擎”。

旧 V2 的世界层、规划层和参考层不删除，但降级为辅助能力；它们不再定义成功标准。新的成功标准是：

- 盲看 1000 字不容易识别为 AI 续写。
- 人物声口稳定。
- 不出现现代分析腔、AI 包装语和解释性总结。
- 长篇硬事实不乱。
- 用户导演意图能被作品内部化，而不是原样进入正文。

## 2. 当前阶段

第一阶段只做红楼专用验证版。

目标不是批量补完后 40 回，而是先把第 81 回打磨到可以进入盲测流程。

## 3. 第一阶段交付

1. 文本清洗与章节切分。
2. 红楼 `style_bible`。
3. 人物声口卡：宝玉、黛玉、宝钗、袭人、王夫人、凤姐、迎春、香菱。
4. `forbidden_words`。
5. 机械风格闸门。
6. 1000 字盲测挑战。
7. 盲测判别器与失败重写循环。
8. 第 81 回单章重写流程。

## 4. 旧 V2 的处理

保留：

- `LiteLLMService`
- session 存储
- Web/API 基础结构
- 已有 quality / benchmark 基础设施
- 已有 V2 模型，作为可选辅助状态

暂停作为主线：

- 重型 LightRAG 图谱驱动
- `WorldModel -> ArcOutline -> ChapterBrief` 的默认主流程
- 后 40 回批量生成

## 5. 后续阶段

第二阶段：通用断更小说版。

- 自动章节切分。
- 自动人物卡。
- 自动伏笔队列。
- 用户导演输入内部化。
- 多轮盲测重写。

第三阶段：长篇规模化。

- 支持 300 万字。
- 支持卷层记忆。
- 支持最近 20 章窗口。
- 支持人物出场检索。
- 支持成本预估。

## 6. 验证

每阶段必须有：

- 单元测试。
- 本地烟测。
- 至少一个真实文本样本。
- 明确失败样本与下一轮修正依据。
